## 📋 Setup Steps

### 1. Environment Configuration

- [ ] **Create .env file**
  ```bash
  # Option 1: Use the helper script (recommended)
  python create_env.py
  
  # Option 2: Manually create .env file
  # Copy the template from README.md and fill in your values
  ```

- [ ] **Configure Supabase**
  - [ ] Create Supabase project at https://supabase.com
  - [ ] Get `SUPABASE_URL` from project settings
  - [ ] Get `SUPABASE_KEY` (anon key) from API settings
  - [ ] Get `SUPABASE_SERVICE_KEY` (service role key)
  - [ ] Get `SUPABASE_DB_URL` from Database settings
  - [ ] Update `.env` file with Supabase credentials

- [ ] **Configure AWS S3**
  - [ ] Create S3 bucket in AWS Console
  - [ ] Create IAM user with S3 access
  - [ ] Get `AWS_ACCESS_KEY_ID` and `AWS_SECRET_ACCESS_KEY`
  - [ ] Update `.env` file with AWS credentials
  - [ ] Configure bucket permissions (CORS if needed)

- [ ] **Configure Main Platform**
  - [ ] Set `MAIN_PLATFORM_API_URL` (default: http://localhost:8000)
  - [ ] Set `MAIN_PLATFORM_API_KEY` if required

- [ ] **Encryption Keys**
  - [ ] Encryption keys auto-generated by `create_env.py`
  - [ ] Or manually generate:
    ```bash
    python -c "import secrets, base64; print('ENCRYPTION_KEY_DERIVATION_SALT=' + secrets.token_urlsafe(32)); print('MASTER_ENCRYPTION_KEY=' + base64.b64encode(secrets.token_bytes(32)).decode())"
    ```

### 2. Dependencies Installation

- [ ] **Install Python dependencies**
  ```bash
  pip install -r requirements.txt
  ```

- [ ] **Verify Python version**
  - Python 3.11+ required (Python 3.14 compatible)
  ```bash
  python --version
  ```

### 3. Database Setup

- [ ] **Run database migrations**
  ```bash
  # Create initial migration (if needed)
  alembic revision --autogenerate -m "Initial migration"
  
  # Apply migrations
  alembic upgrade head
  ```

- [ ] **Verify database connection**
  - Start the service and check logs for database connection status

### 4. Docker Setup (Optional)

- [ ] **Create Docker network**
  ```bash
  docker network create myhigh5-network
  ```

- [ ] **Build and run with Docker**
  ```bash
  docker-compose up -d
  ```

### 5. Service Startup

- [ ] **Start the service**
  ```bash
  # Development
  python main.py
  
  # Or with uvicorn
  uvicorn main:app --host 0.0.0.0 --port 8001 --reload
  ```

- [ ] **Verify service is running**
  - Check http://localhost:8001/health
  - Should return: `{"status": "healthy", "service": "feed-microservice", "version": "1.0.0"}`

### 6. API Documentation

- [ ] **Access Swagger UI**
  - Open http://localhost:8001/docs
  - Verify all endpoints are visible

- [ ] **Access ReDoc**
  - Open http://localhost:8001/redoc

### 7. Testing

- [ ] **Test health endpoint**
  ```bash
  curl http://localhost:8001/health
  ```

- [ ] **Test authentication**
  - Get JWT token from main platform
  - Test protected endpoint with Authorization header

- [ ] **Test encryption key generation**
  - POST /api/v1/keys/generate
  - Verify keys are created and encrypted

- [ ] **Test message encryption**
  - Send a test message
  - Verify it's encrypted in database
  - Verify only recipient can decrypt

## 🚨 Common Issues

### Database Connection Failed
- **Solution**: Check `SUPABASE_DB_URL` in `.env` file
- **Note**: Service will start with warning if database fails (for development)

### Missing Dependencies
- **Solution**: Run `pip install -r requirements.txt`
- **Note**: Python 3.14 requires updated httpx/httpcore (already in requirements.txt)

### Port Already in Use
- **Solution**: Change `PORT` in `.env` file or stop other service on port 8001

### Docker Network Not Found
- **Solution**: Run `docker network create myhigh5-network`
